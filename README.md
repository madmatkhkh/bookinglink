# نوبت‌لینک (bookinglink) — پلتفرمِ نوبت‌دهیِ چند‌مستاجریِ چند‌نیچی

پلتفرمِ SaaS که به هر متخصص/کسب‌وکار یک صفحه‌ی نوبت‌دهیِ برندشده‌ی اختصاصی می‌دهد.
یک codebase، یک دیتابیس، یک deployment؛ هر مشتری = یک ردیف در جدولِ `tenants`.

## معماریِ سه‌لایه
- **لایه ۱ — هسته (کد):** نوبت، تقویمِ جلالی، پرداخت، پرونده، پنل، منابع. برای همه‌ی نیچ‌ها یکی.
- **لایه ۲ — تمپلیتِ نیچ (دیتا):** جدولِ `niches`. هر نیچ برچسب‌ها، فیلدهای پرونده، ماژول‌های پیش‌فرض، رنگ و سرویس‌های نمونه را تعریف می‌کند.
- **لایه ۳ — ماژول‌ها (`tenant_features`):** قابلیت‌های اختیاری/پولی، بعضی مشترک بعضی خاصِ نیچ.

**قانونِ طلایی:** ساختار = کد، محتوا و قابلیت = دیتا.

## قابلیت‌ها
- **چند‌منبعی (multi-resource):** هر tenant می‌تواند چند پرسنل داشته باشد. مشتری هنگامِ رزرو یا یک نفرِ مشخص را انتخاب می‌کند یا «هرکه آزاد بود». تک‌نفره‌ها یک منبعِ پیش‌فرض دارند.
- **دو نیچِ آغازین:** روانشناسی/روان‌پزشکی و سالنِ زیبایی.
- **دامنه‌ی اختصاصی:** ستونِ `custom_domain` روی tenant آماده است (verify در فاز بعد).
- **پرونده‌ی مشتری نیچ‌محور:** همان تبِ پرونده، فیلدهای متفاوت به‌ازای نیچ.

## استک
Next.js 14 (App Router) · TypeScript · Tailwind · Supabase (Postgres + RLS)

## راه‌اندازی
1. اسکیما: کلِ `supabase-schema.sql` را در SQL Editor سوپابیس اجرا کنید (دو نیچِ اولیه seed می‌شوند).
2. `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `SUPER_SECRET` (کلمه‌ی عبورِ پنلِ سوپرادمین)
   - `AUTH_SECRET` (رشته‌ی تصادفی برای امضای نشست)
3. `npm install && npm run dev`
4. سوپرادمین: `/super` → با `SUPER_SECRET` وارد شوید، tenant بسازید (نیچ را انتخاب کنید).
5. هر tenant: صفحه‌ی عمومی `/<slug>`، پنلِ متخصص `/<slug>/panel`.

## ساختارِ دیتابیس (خلاصه)
`niches` (تمپلیت‌ها) · `tenants` (+custom_domain, niche_key) · `tenant_profiles` · `resources` (پرسنل؛ +phone/owner_session برای ورودِ مستقلِ کارمند) · `services` + `service_resources` · `weekly_schedules` + `schedule_overrides` (per-resource) · `bookings` (+resource_id) · `client_records` (پرونده) · `otps` · `tenant_features`

## نکاتِ فنی
- اسلات ذخیره نمی‌شود، محاسبه می‌شود: (برنامه‌ی هفتگیِ منبع + استثناها) − رزروهای زنده. ضامنِ همزمانی، unique index روی `(tenant, resource, date, time)`.
- اعدادِ ذخیره‌شده لاتین، نمایش فارسی.
- OTP فعلاً کدِ تست را در پاسخِ API برمی‌گرداند (`dev_code`) — تا اتصالِ ماژولِ پیامک.
- امنیت: RLS روی همه‌ی جدول‌ها، دسترسی فقط از سمتِ سرور با `service_role`.
