# نوبت‌لینک (bookinglink)

پلتفرم SaaS چندمستاجری/چندنیچی نوبت‌دهی و مدیریت کسب‌وکار. هر مشتری یک ردیف در
جدول `tenants` است؛ یک codebase، یک دیتابیس، یک deployment.

- **دیپلوی:** nobatlink.com
- **استک:** Next.js 14.2 (App Router) · TypeScript · Tailwind · Supabase (Postgres)
- **زیرساخت جانبی:** Cloudflare R2 (فایل)، sms.ir (پیامک)، Resend (ایمیل)،
  درگاه پرداخت (از طریق رله‌ی آی‌پی ثابت)

## راه‌اندازی محلی

```bash
npm install
cp .env.example .env.local   # مقادیر را پر کن
npm run dev
```

حداقل متغیرهای لازم برای اینکه build رد شود: `NEXT_PUBLIC_SUPABASE_URL`،
`SUPABASE_SERVICE_ROLE_KEY`، `SUPER_SECRET`، `AUTH_SECRET`. بقیه اختیاری‌اند و
نبودنشان فقط قابلیت مربوطه را بی‌صدا غیرفعال می‌کند (fail-open). توضیح تک‌تک در
`.env.example`.

دیتابیس تازه: کل `schema.sql` را در Supabase → SQL Editor اجرا کن.
⚠️ آن فایل اول همه‌چیز را drop می‌کند — فقط روی دیتابیس بدون داده‌ی واقعی.

## ساختار

```
src/app/[slug]/          صفحات هر tenant (عمومی، پنل مراجع، پنل متخصص)
src/app/api/t/[slug]/    API هر tenant — psy/* سمت مراجع، panel/* سمت متخصص
src/app/api/super/       سوپرادمین
src/app/api/cron/        زمان‌بندها (یادآوری، بیلینگ)
src/app/super/           پنل سوپرادمین
src/lib/                 منطق مشترک (auth، slots، ledger، modules، psy، ...)
src/components/          کامپوننت‌های مشترک
```

پنل متخصص در `src/app/[slug]/panel/` است: `page.tsx` فقط روتر نیچ است و
**هرگز نباید محتوای پنل داخلش برود**؛ تب‌های سنگین در `panel/modules/*` جدا
شده‌اند.

## قراردادهای کاری

جزئیات کامل در `CLAUDE.md` § «قراردادهای کاری». خلاصه:

- قبل از هر تحویل: `npx next build` باید **exit 0** بدهد.
- **ارقام همه‌جا لاتین** — در UI، در دیتابیس، و در مقادیر پیش‌فرض ستون‌ها.
  (نرمال‌سازی ورودی کاربر از رقم فارسی به لاتین دست‌نخورده می‌ماند.)
- **بدون اعراب** (فتحه/کسره/ضمه/تنوین) در هیچ متن فارسی‌ای، حتی کامنت کد.
- تقویم **جلالی** در همه‌جا.
- تغییر اسکیما فقط **additive** — بازسازی مخرب از migration 0019 منسوخ است.
