## ریفکتور پنل (رفتار runtime صفر تغییر)

`PsychologyAdmin.tsx`: **3540 → 2318 خط**. بلوک تنظیمات: 900+ → 128 خط، و
وابستگی‌اش به والد از **72 شناسه به 39**.

مسیر کامل این فایل از ابتدای فاز 4: 5700 → 2318.

**چرا امن بود:** بلوک‌های زیرتب‌های تنظیمات در والد درهم بودند
(profile → payments → profile → appearance → ...)، ولی چون هرکدام پشت گارد
`settingsSubTab === X` است و هم‌زمان فقط یکی رندر می‌شود، گروه‌کردنشان هیچ
تغییر بصری نمی‌دهد. ترتیب داخل هر زیرتب عینا حفظ شد.

`intakeForm` عمدا در والد ماند — نوار ذخیره‌ی مشترک باید تغییراتش را ببیند،
وگرنه دکمه‌ی ذخیره ظاهر نمی‌شد و تغییرات کاربر بی‌صدا از بین می‌رفت.

⚠️ **موقع تست، کل تب تنظیمات را دور بزن:**
- **پروفایل:** نام/عنوان/بج‌ها، آپلود و برش عکس، سوییچ حضوری/آنلاین،
  کانال‌های آنلاین، مکان‌های حضوری
- **ظاهر:** انتخاب تم و ذخیره‌اش
- **پرداخت‌ها:** شماره کارت‌ها، شبا، روش‌های پرداخت
- **قیمت‌گذاری:** قیمت آنلاین/حضوری، مالیات، دقیقه‌ی اضافه، کدهای تخفیف
- **شرایط و مقررات:** متن و سوییچ
- **فرم رزرو:** انتخاب سوال، آکاردئون، درگ‌اند‌دراپ، زیرسوال شرطی
- و مخصوصا: **سوییچ بین دکترها** + اینکه نوار «ذخیره‌ی تغییرات» در هر شش
  زیرتب سر جایش ظاهر شود

## خلاصه‌ی پنج فیکس

1. **کد تخفیف — پیام موفقیت کهنه:** تعویض نوع جلسه (حضوری/آنلاین) تخفیف
   واقعی را پاک می‌کرد ولی پیام «کد اعمال شد» می‌ماند. رفع: `key={mode}`
   روی `DiscountCodeField`.
2. **کانال پروتکل:** جلسه‌ی زیر یک پروتکل همه‌ی کانال‌های فعال دکتر را
   نشان می‌داد، نه فقط کانال انتخابی پروتکل. رفع: `pkgChannel` در
   `SessionCard`.
3. **کانال جلسه‌ی مستقل:** جلسات بدون پروتکل کانال مشخصی نداشتند. رفع:
   چوزر کانال قبل از «انتخاب زمان جلسه» + ستون `meet_channel`.
4. **کد تخفیف — علت اصلی گزارش دوم:** `psy/pay-online` اگر اعتبارسنجی
   سمت سرور کد تخفیف شکست می‌خورد (مثلا کد بین پیش‌نمایش و لحظه‌ی واقعی
   پرداخت منقضی/غیرفعال شده بود)، هیچ خطایی برنمی‌گرداند — بی‌سروصدا با
   قیمت کامل ادامه می‌داد. مسیر کارت‌به‌کارت از اول درست بود (خطا
   برمی‌گرداند)؛ حالا `pay-online` هم همین رفتار را دارد. **این احتمالا
   ریشه‌ی واقعی چیزی است که دیدید** — دفعه‌ی بعد اگر کد تخفیف کار نکرد،
   یک پیام خطای روشن می‌بینید (مثلا «این کد دیگر فعال نیست») به‌جای
   قیمت کامل بی‌صدا.
5. **فلش پیام «هنوز ثبت نشده»:** کارت جلسه‌ی پروتکل یک لحظه پیام هشدار
   نشان می‌داد، بعد خودکار به دکمه‌ی کانال درست تبدیل می‌شد — چون دیتای
   کانال‌های دکتر با یک fetch جدا می‌رسید و کد قبل از رسیدنش قضاوت
   می‌کرد. رفع: تا `settings.loaded` نرسیده، اسکلتون نشان داده می‌شود،
   نه پیام.

`npx next build` روی هر پنج فیکس → exit 0. اسکن رقم فارسی/اعراب: صفر.
